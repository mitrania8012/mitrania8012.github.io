This is where the majority of your text is going to be. <!DOCTYPE html> declares a document to be in html. <html> sets your computer up for reading html. <head> is a container for metdata, such as the page title. The <title> element is used to title your page. This title shows up on the tab when you open it. <body> is what shows up on your page, visible to the page viewer. To set up a document, use this layout:

<!DOCTYPE html>
<html>
<head>
<title> Page Title </title>
<style>
Style elements can go here, but are better in a seprate CSS sheet.
</style>
</head>
<body>

All of your visible code goes here, including headings, paragraphs, pictures, etc.

</body>
</html>

What if you want to put something in the body, like a note for yourself, and don’t want it to show up in the final website? Simple. Comment it out using

Another thing you can do in HTML is make a table.
To make a table, follow this format:
<table style="width:100%">
<tr>
<th>Collumn #1 Title</th>
<th>Collumn #2 Title</th>
<th>Collumn #3 Title</th>
</tr>
<tr>
<td>First row, first spot</td>
<td>First row, second spot</td>
<td>First row, third spot</td>
</tr>
<tr>
<td>Second row, first spot</td>
<td>Second row, second spot</td>
<td>Second row, third spot</td>
</tr>
</table>
So what do these things mean? <table style="width:100%"> tells the computer you are making a table that spans 100% of the width. <tr> is used to divide different rows. <th> is for the titles of the collums. It bolds and centers the text. Finally, <td> defines where different words/numbers become different elements in a table.

Name Age
Jill 23
Eve 13

Adding lines in between the boxes is a style element. To add them, put the following in bewteen the style tags at the beginning:
table, th, td {
border: 1px solid black;
border-collapse: collapse;
}
The collapse is optional. Without it, there are two lines instead of one.

Finally, there is the <div> element. <div> is a block element. When you use it, the text jumps to a new line kind of like with <br>. It is used like this:
<div> Text </div>
<div> More text </div>
The final result would be:

Text
More text

So why do we use it instead sometimes?
With <div>, you can apply style to large amounts of text at once. Although it is best to do style using CSS, you can also apply it directly into HTML. Let's look at an example.

Hello World!

Colors!

HTML!

This can be done one of two ways:
<p style= "color:blue;"> Hello World! </p>
<p style= "color:blue;"> Colors! </p>
<p style= "color:blue;"> HTML! </p>
Or, the much simpler, easier way:
<div style= "color:blue;">
<p> Hello World! </p>
<p> Colors! </p>
<p> HTML! </p>
</div>

Both of these will have the same result, but one is easier. As you can see, <div> is a very useful block element. It can be used to apply style to large batches of text, and even just as a divider for two separate parts of something.
